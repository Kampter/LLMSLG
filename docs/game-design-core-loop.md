# 星际探索 SLG — 核心循环设计文档

## 1. 概述

本文档定义游戏的最小可玩闭环：

> **玩家经营一个轨道基地，采集能源与矿物，建造飞船，派遣飞船前往星球采集资源，再用资源扩张基地与舰队。**

核心体验是**能源现金流管理**与**空间调度决策**的结合。

---

## 2. 核心循环

```
┌─────────────────────────────────────────────────────────────┐
│  反应堆产出能源 ──► 基地储备能源                              │
│         │                                              │    │
│         │                                              │    │
│         ▼                                              ▼    │
│    派遣飞船（消耗能源）                              升级    │
│         │                                        反应堆/船坞 │
│         │                                              │    │
│         ▼                                              │    │
│    飞船抵达目标星球 ◄───────────────────────────────────┘    │
│         │                                                   │
│         ▼                                                   │
│    采集矿物（消耗时间）                                      │
│         │                                                   │
│         ▼                                                   │
│    返航卸载矿物 ──► 矿物储备 ──► 建造飞船 / 升级设施          │
└─────────────────────────────────────────────────────────────┘
```

---

## 3. 空间架构

### 3.1 层级

```
星系 (Galaxy)
├── 恒星
│   └── 提供星系级能源背景增益
│
├── 星球 A
│   ├── 轨道 ── 基地（玩家起点）
│   │   ├── 反应堆（自动产出能源）
│   │   ├── 船坞（停靠飞船，有容量上限）
│   │   └── 飞船编队
│   └── 地表 ── 矿物资源点
│
├── 星球 B ── 富矿，无基地
├── 星球 C ── 气态，能源丰富
└── 小行星带 ── 散矿，采集快但总量少
```

### 3.2 关键设定

- **基地只能建在星球轨道上**，不能建在星球表面
- 所有资源采集都需要飞船往返
- 初始阶段只有一个基地

---

## 4. 资源系统

### 4.1 双资源设计

| 资源     | 来源                      | 采集方式               | 本质                 |
| -------- | ------------------------- | ---------------------- | -------------------- |
| **能源** | 基地反应堆 + 恒星光照增益 | **自动产出**，无需飞船 | 时间的函数（现金流） |
| **矿物** | 星球地壳 / 小行星         | **必须派飞船降落采集** | 空间的函数（资本）   |

**设计意图**：能源是"在线/离线都在增长"的背景资源；矿物是"必须主动操作"的前置资源。两者缺一不可。

### 4.2 资源属性

| 属性         | 能源                        | 矿物                         |
| ------------ | --------------------------- | ---------------------------- |
| 是否自动增长 | 是                          | 否                           |
| 是否可透支   | 是（触发降载惩罚）          | 否                           |
| 存储上限     | 有（由反应堆/仓储等级决定） | 有（由船坞/仓储等级决定）    |
| 离线收益     | 能源继续产出                | 矿物不增（无飞船在途则不增） |

---

## 5. 飞船行为链

### 5.1 状态机

```
停靠基地 ──► 装填能源 ──► 航行中 ──► 抵达轨道
                                          │
                                          ▼
                                      降落中（消耗时间 + 能源）
                                          │
                                          ▼
                                      采集中（消耗时间，产出矿物）
                                          │
                                          ▼
                                      升空中（消耗能源 > 降落）
                                          │
                                          ▼
                                      航行中 ──► 抵达基地 ──► 卸载矿物 ──► 停靠
```

### 5.2 能源消耗点

| 阶段         | 消耗                          | 备注                    |
| ------------ | ----------------------------- | ----------------------- |
| 装填         | 按飞船能源舱容量填充          | 从基地能源储备扣除      |
| 航行（去程） | `距离 × 单位距离能耗`         | 距离越远消耗越大        |
| 降落         | `星球重力系数 × 基础值`       | 大质量星球消耗更高      |
| 采集         | 无（时间成本）                | 纯时间等待              |
| 升空         | `星球重力系数 × 基础值 × 1.5` | 模拟逃逸速度 > 降落速度 |
| 航行（返程） | `距离 × 单位距离能耗`         | 与去程相同              |

**关键约束**：飞船能源舱容量有限，远航需精确计算往返能源，否则可能 stranded（滞留）。

### 5.3 采集产出

```
单次采集产出 = 基础采集速率 × 星球富饶度 × 采集时间
```

| 采集点类型   | 富饶度 | 特点                                |
| ------------ | ------ | ----------------------------------- |
| 近地小行星带 | 低     | 距离近、往返快、适合早期            |
| 本星球地表   | 中     | 距离为零（无需航行），但需降落/升空 |
| 远轨道气态云 | 中     | 采集能源而非矿物（特殊资源点）      |
| 其他星球     | 高     | 距离远、往返慢、但单次收益大        |

## 6. 决策点

每个游戏周期（30-60秒为一个"回合"感），玩家需要决策：

### 6.1 能源分配

```
当前能源储备
    ├── 造船（消耗大量矿物 + 少量能源）
    ├── 派遣飞船（消耗航行能源）
    ├── 升级反应堆（消耗大量矿物，一次性）
    └── 升级船坞（消耗大量矿物，提升容量上限）
```

### 6.2 飞船调度

```
N艘可用飞船
    ├── 派 K 艘去近地小行星（快、稳、少）
    ├── 派 M 艘去本星球地表（无航行、纯降落成本）
    └── 派 P 艘去远星（慢、险、多）
```

### 6.3 科技/升级优先级

| 升级方向   | 效果             | 前置条件               |
| ---------- | ---------------- | ---------------------- |
| 反应堆升级 | 能源产出 +X%     | 矿物投入               |
| 船坞扩建   | 飞船容量上限 +N  | 矿物投入               |
| 飞船能源舱 | 单次航行距离提升 | 矿物投入               |
| 跃迁引擎   | 解锁跨星球航行   | 大量矿物（阶段性目标） |

---

## 7. 成长与扩张路径

### 7.1 阶段一：单基地自给自足（教学期）

- 1个基地，1艘飞船
- 3个采集点可用（小行星、本星球、气态云）
- 目标：学会能源现金流管理，积累首批矿物

### 7.2 阶段二：舰队扩张（增长期）

- 船坞容量提升，拥有 3-5 艘飞船
- 同时调度多艘飞船去不同采集点
- 目标：矿物收入进入指数增长

### 7.3 阶段三：跨星球扩张（破局点）

- 研发跃迁引擎（一次性大量矿物投入）
- 建造星际货船（不同于近程采矿船）
- 在新星球轨道建立前哨站

### 7.4 阶段四：物流网络（后期）

- 不是所有星球都能自给自足
- 部分星球只有矿物，没有反应堆，需从主基地运能源过去
- 建立星际物流航线

---

## 8. 瓶颈与失败设计

无需敌人，纯经济系统即可产生有意义的约束：

| 瓶颈         | 触发条件                 | 效果                                           |
| ------------ | ------------------------ | ---------------------------------------------- |
| **能源赤字** | 能源 < 0 持续 5 秒       | 反应堆强制降载 50%，产出减半，恢复缓慢         |
| **船坞满载** | 当前飞船数 = 容量上限    | 无法造船，矿物堆积，增长停滞                   |
| **滞留风险** | 飞船能源不足以返航       | 飞船进入"求援"状态，需派救援船（消耗更多能源） |
| **远距陷阱** | 一次性派遣过多飞船去远矿 | 能源短期透支，基地运转停滞                     |

---

## 9. MVP 最简范围

如果只做"能闭环的最小版本"，保留以下内容即可：

| 保留                                 | 砍掉（留待后续） |
| ------------------------------------ | ---------------- |
| 1 个基地                             | 多星球扩张       |
| 2 种资源（能源、矿物）               | 物流网络         |
| 1 种通用飞船                         | 飞船细分类型     |
| 3 个采集点（小行星、本星球、气态云） | 其他星球         |
| 反应堆 + 船坞 两个升级方向           | 更多科技树       |
| 能源赤字 + 船坞满载 两个瓶颈         | 滞留救援机制     |

**MVP 玩家体验**：

> "我有 100 能源和 1 艘船。我该派它去小行星快速回来，还是去本星球地表省航行能源？我的能源够造船还是该先升级反应堆？"

---

## 10. 设计决策确认

| 问题           | 决策                                                                                    |
| -------------- | --------------------------------------------------------------------------------------- |
| **时间尺度**   | 实时连续运行，通过不同距离的往返时间创造"自然决策节拍"（近程 30-60 秒，远星 2-10 分钟） |
| **离线收益**   | 完全正常产出，飞船在途任务也正常执行                                                    |
| **视觉呈现**   | **2D 俯视轨道图** —— 星系平面展开，星球、轨道、飞船、采集点以图标形式呈现               |
| **数据持久化** | **持续世界** —— 服务器端持续运行，玩家随时登入看到当前状态                              |
| **多玩家互动** | **异步观光** —— 玩家可以查看其他玩家的星系状态，无直接交互（战斗、贸易等）              |
